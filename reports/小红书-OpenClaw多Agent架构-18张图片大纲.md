## 图片 11：Agent 间通信机制

### 内容要点
- **核心机制**: sessions_send
- **工作原理**:
  1. Main Agent 发送消息
  2. 指定目标 Agent（sessionKey）
  3. 目标 Agent 接收并处理
  4. 返回结果给 Main
- **关键参数**:
  - `sessionKey`: 目标 Agent 标识
  - `message`: 发送的消息内容
  - `timeoutSeconds`: 超时时间
- **特点**: 
  - 异步通信
  - 支持超时
  - 可追踪状态

### 架构图描述
```
[Main Agent]
     ↓
sessions_send({
  sessionKey: "coding",
  message: "写一个登录功能"
})
     ↓
[Coding Agent]
     ↓
处理并返回结果
     ↓
[Main Agent]
```

### 设计建议
- **配色**: 白色背景 + 蓝色箭头
- **布局**: 流程图式，从左到右
- **视觉**: 消息气泡、箭头、Agent 图标
- **字体**: 代码部分用等宽字体

## 图片 12：并行协作原理

### 内容要点
- **核心概念**: 同时向多个 Agent 发送任务
- **实现方式**: 同一 function_calls 块中多个 sessions_send
- **优势**:
  1. 效率提升（并行执行）
  2. 专业分工（各司其职）
  3. 质量保证（专家审核）
- **适用场景**:
  - 内容生产（research + creative + design）
  - 代码开发（coding + qa + ops）
  - 项目管理（planning + tracking + reporting）

### 架构图描述
```
[Main Agent]
     ↓
function_calls:
  ├─ sessions_send(research, "调研...")
  ├─ sessions_send(creative, "写文案...")
  └─ sessions_send(design, "设计...")
     ↓
[Research] [Creative] [Design]
（并行执行）
     ↓
[Main Agent] 汇总结果
```

### 设计建议
- **配色**: 白色背景 + 多色箭头（每个 Agent 不同色）
- **布局**: 扇形展开式
- **视觉**: 并行箭头、同时执行效果
- **字体**: 清晰标注并行

## 设计风格总结

### 配色方案
- **主色**: 渐变蓝/紫色（科技感）

- **背景**: 白色为主，简洁清爽
- **文字**: 深灰/黑色，易读

### 字体方案
- **标题**: 思源黑体 Bold / PingFang SC Semibold
- **正文**: 思源黑体 Regular / PingFang SC Regular
- **代码**: JetBrains Mono / Fira Code（等宽字体）
- **数字**: DIN / Roboto Mono

### 布局原则
- **简洁**: 每张图片聚焦一个核心概念
